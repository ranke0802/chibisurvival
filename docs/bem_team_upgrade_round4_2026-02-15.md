# bem 팀 고도화 회의 및 실행 결과 (Round 4)

- 일시: 2026-02-15
- 참석: QA, 개발, 게임디자인, 그래픽디자인, 사운드디자인
- 목적: 캐릭터 움직임 자연화 + 보스 패턴별 타격감 강화

## 1) 직군별 논의 요약
1. 개발
- 단일 PNG 기반 캐릭터는 보행 프레임 정보가 없어 움직임이 뻣뻣하게 보임.
- 해결: 전사/궁수에 대해 모션 시트를 추가하고 렌더러는 시트 우선 방식으로 전환.

2. 게임디자인
- 보스 패턴이 같아 보이면 회피 학습 재미가 떨어짐.
- 해결: 원/선/부채꼴 패턴마다 다른 연출(이펙트/예고음/타격감)을 부여.

3. 그래픽디자인
- 스프라이트 투명 여백이 큰 경우 실제 출력 비율이 어색해짐.
- 해결: 렌더 단계에서 자동 트리밍을 적용해 체감 크기 왜곡을 완화.

4. 사운드디자인
- 보스 패턴 전조 사운드가 없으면 위험 인지 타이밍이 늦음.
- 해결: 텔레그래프 생성 시 패턴별 저강도 예고음을 추가.

5. QA
- 결과 검증 항목:
  - 캐릭터 이동 애니메이션의 자연스러움
  - 보스 패턴별 피드백 차별화
  - 빌드 안정성

## 2) 실행 항목
1. 전사/궁수 모션 시트 생성
- 생성 파일:
  - `public/assets/characters/warrior_motion_sheet.png`
  - `public/assets/characters/archer_motion_sheet.png`
- 포맷: 8열 x 2행(Idle/Walk) 시트

2. 플레이어 렌더링 개선
- 캐릭터별 모션 시트 우선 사용(`warrior/mage/archer`)
- 단일 PNG 사용 시 자동 트리밍 + 픽셀 스냅 적용
- 반영 파일: `src/game/renderer.ts`

3. 보스 패턴별 타격감 분리
- `line`: 오렌지 빔 연출 + 레이저 계열 예고음
- `cone`: 부채꼴 베기 연출 + 블레이드 계열 예고음
- `circle`: 낙뢰 연출 + 번개 계열 예고음
- 공통 피격 로직은 함수화해 유지보수성 개선
- 반영 파일: `src/game/engine.ts`, `src/game/renderer.ts`

## 3) 검증
1. 빌드 검증
- 명령: `npm run build`
- 결과: 성공

2. QA 포인트
- 전사/궁수 이동 시 프레임 전환으로 보행감이 증가했는지
- 보스 패턴별 위험 인지가 시각/청각으로 분명해졌는지
- 기존 설정/HUD 동작 회귀 없는지
