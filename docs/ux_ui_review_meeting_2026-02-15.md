# UX/UI 평가 및 반영 회의록

- 일시: 2026-02-15
- 범위: 플레이어 체감 UX/UI, 정보 구조, 피로도, 전투 가독성
- 빌드 검증: `npm run build` 통과 (TypeScript + Vite 정상 빌드)

## 담당자 지정
1. UX 리드: 김지윤
2. UI 디자이너: 박민서
3. 게임 기획: 이도현
4. 접근성 QA: 최수빈
5. 개발 담당: Codex

## 사전 평가 결과
1. UX 리드
- 문제: 전투 중 정보 우선순위가 분산되어 목표 인지가 느림.
- 제안: 중앙 HUD에 "현재 목표"를 지속 노출하고, 스테이지 전환 흐름을 자동화.

2. UI 디자이너
- 문제: 우측 HUD가 숫자 나열형이라 스캔 속도가 느림.
- 제안: KPI 칩 + 스킬 칩 구조로 분리하고, 모바일에서 밀도 축소.

3. 게임 기획
- 문제: 보스 출현 직전 긴장감 전달이 약함.
- 제안: 보스 출현 10초 전 경고 배너 추가.

4. 접근성 QA
- 문제: 저체력/피격 상황의 시각 피로가 누적될 수 있음.
- 제안: 점멸보다 비점멸형 위험 신호를 우선 사용, 버튼 포커스 가시성 강화.

5. 개발 담당
- 문제: 스테이지 클리어 후 수동 입력 대기는 흐름 단절을 유발.
- 제안: 자동 전환 카운트다운 + 즉시 진행 버튼 병행.

## 회의 결론
- 우선순위 1: 점멸/플래시 최소화 유지.
- 우선순위 2: 목표 정보와 전환 흐름을 한눈에 보이게 변경.
- 우선순위 3: 초반 이탈 방지용 간단 온보딩 카드 제공.
- 우선순위 4: 모바일 정보 밀도 감축.

## 반영한 결과물
1. 자동 전환 흐름
- 보스 처치 후 다음 스테이지 자동 시작 카운트다운 추가.
- 수동 시작 버튼은 유지.
- 반영: `src/game/engine.ts`, `src/game/types.ts`, `src/App.tsx`

2. HUD 재구성
- 우측 HUD를 KPI/스킬 칩 구조로 정리.
- 스킬마다 Lv + 쿨다운 표시.
- 반영: `src/App.tsx`, `src/styles.css`

3. 경고 및 안내
- 보스 출현 10초 전 경고 배너 추가.
- 저체력 위험 상태 시 비점멸 테두리 오버레이 추가.
- 초반 온보딩(이동/목표/강화 추천) 카드 추가.
- 반영: `src/App.tsx`, `src/styles.css`

4. 접근성
- 버튼 `focus-visible` 강조 추가.
- 반영: `src/styles.css`

## 후속 액션
1. Gemini 에셋 반영 후 64px 가독성 기준으로 1차 컷오프.
2. 보스 패턴 텔레그래프(위험 범위) UI 추가.
3. 스킬 이펙트 강도 옵션(낮음/기본) 제공 검토.
