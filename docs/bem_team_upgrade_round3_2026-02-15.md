# bem 팀 고도화 회의 및 실행 계획 (Round 3)

- 일시: 2026-02-15
- 참석: QA, 개발, 게임디자인, 그래픽디자인, 사운드디자인
- 전제: `public/assets/characters/chibi.jpg` 기반 마법사 스타일을 기준 비주얼로 유지

## 1) 직군별 핵심 의견
1. 개발
- 현재 `App -> Renderer` 옵션 연결 누락으로 빌드 실패가 발생하고 있어 우선 복구 필요.
- 설정이 늘어날수록 프리셋이 있어야 실사용 속도가 빨라짐.

2. 게임디자인
- 보스 텔레그래프는 시인성 옵션이 있어야 난이도 체감 편차를 줄일 수 있음.
- 사용자별 선호(몰입형/가독형)를 즉시 전환할 수 있는 전투 설정 프리셋 필요.

3. 그래픽디자인
- 캐릭터 동작의 어색함은 일부 단일 PNG의 과도한 여백/비율 문제에서 발생.
- 마법사는 시트 애니메이션 우선, 단일 스프라이트는 자동 트리밍으로 형태 왜곡을 줄여야 함.

4. 사운드디자인
- 스킬 과밀 구간에서 가시성 옵션과 함께 프리셋 기반 볼륨 세팅이 유효함.
- 마스터/BGM/SFX를 개별 조절하되 진입 장벽을 낮추기 위해 프리셋 버튼 제공 권장.

5. QA
- "설정 저장 후 복귀" 플로우에 프리셋/고대비 옵션까지 포함되어야 회귀 테스트 범위가 명확해짐.
- 빌드 통과 + 인게임 설정 반영 여부를 이번 라운드 DoD로 지정.

## 2) 합의된 우선순위
1. 빌드 실패 원인 제거(`RenderOptions` 연동 완성)
2. 실전 가독성 옵션 추가(고대비 텔레그래프)
3. 설정 프리셋/초기화 추가(쾌적/기본/강렬/초기화)
4. 캐릭터 렌더링 자연화(마법사 모션 우선 + 스프라이트 트리밍)

## 3) 이번 라운드 DoD
1. `npm run build` 통과
2. 설정 모달에서 프리셋 즉시 적용 및 저장 반영
3. 고대비 텔레그래프 ON/OFF가 인게임에서 즉시 반영
4. 캐릭터 비율 찌그러짐/여백 문제 완화 확인

## 4) 실행 결과
1. 설정/렌더 연동 복구 및 기능 확장
- `highContrastTelegraphs` 옵션을 `LiveSettings`에 추가하고 렌더러에 즉시 전달.
- 설정 프리셋 3종(쾌적/기본/강렬)과 기본값 복원 버튼을 추가.
- 반영 파일: `src/App.tsx`, `src/styles.css`

2. 캐릭터 렌더링 보정
- 마법사는 모션 시트를 우선 사용하도록 렌더 우선순위 조정.
- 단일 PNG 스프라이트(캐릭터/몬스터) 렌더 시 투명 여백 자동 트리밍 적용.
- 픽셀 스냅(좌표 반올림)으로 이동 시 미세 떨림 완화.
- 반영 파일: `src/game/renderer.ts`

3. 마법사 포트레이트/기본 스프라이트 정리
- `chibi_sheet` 기반으로 `chibi_portrait.png`, `mage.png`를 재정렬하여 과도한 여백 축소.
- 반영 파일:
  - `public/assets/characters/chibi_portrait.png`
  - `public/assets/characters/mage.png`

## 5) 검증
1. 빌드 검증
- 명령: `npm run build`
- 결과: 성공 (TypeScript/Vite 모두 통과)

2. QA 확인 포인트
- 설정 모달에서 프리셋 클릭 시 슬라이더/토글 값 동기화.
- 고대비 텔레그래프 토글 시 보스 위험영역 컬러가 즉시 변경.
- 마법사 이동 시 모션 프레임 기반 보행이 적용되고, 캐릭터 크기 왜곡이 감소.
